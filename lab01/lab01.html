¿Cuál es la diferencia entre Internet y la World Wide Web?

La Web en la World Wide Web no se refiere a una red de computadoras conectadas, sino a una red de información conectada por hipervínculos. La red vinculada de computadoras, Internet, es la base sobre la cual se ha construido la Web y dependemos de Internet para darnos acceso a esa Web y permitirnos agregarla. Sin Internet, no hay World Wide Web. Dicho esto, la Web es la parte más popular de Internet, por lo que es fácil ver por qué la persona promedio considera que los términos son sinónimos.

 ¿Cuáles son las partes de un URL?

Las direcciones Web o «URLs» son la base de Internet y una forma sencilla de identificar y encontrar cualquier cosa que buscamos en la red como páginas web, fotos y videos.  ¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?
  • GET

    • Solicita una representación del recurso especificado. Solicitudes que usan GET solo debe recuperar datos.

  • HEAD

    • Pide una respuesta idéntica a la de una petición GET, pero sin el cuerpo de la respuesta.

  • POST

    • Se utiliza para enviar una entidad a un recurso en específico, causando a menudo un cambio en el estado o efectos secundarios en el servidor.

  • PUT

    • Reemplaza todas las representaciones actuales del recurso de destino con la carga útil de la petición.

  • PATCH

    • Es utilizado para aplicar modificaciones parciales a un recurso.

  • DELETE

    • Borra un recurso en específico.

Cita:

Métodos de petición HTTP - HTTP | MDN. (2023, July 24). MDN Web Docs. https://developer.mozilla.org/es/docs/Web/HTTP/Methods
 ¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

Al enviar un formulario HTML, se debe utilizar el método HTTP GET o POST. La elección entre GET y POST depende de la naturaleza de los datos que se están enviando. El método GET se utiliza para enviar datos no confidenciales o para recuperar información del servidor, ya que los datos se envían a través de la URL y son visibles en ella. Por otro lado, el método POST se emplea para enviar datos confidenciales o archivos, ya que los datos se envían en el cuerpo de la solicitud y no son visibles en la URL, lo que los hace más seguros. En resumen, se debe usar GET para datos no confidenciales y POST para datos confidenciales al enviar un formulario HTML

Cita:

Sending form data - Aprende desarrollo web | MDN. (n.d.). MDN Web Docs. https://developer.mozilla.org/es/docs/Learn/Forms/Sending_and_retrieving_form_data
 ¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

Al escribir la dirección URL en el navegador, este envía una petición GET al servidor para recuperar la página solicitada. La petición GET es la más común y se utiliza para obtener datos del servidor sin realizar cambios en el estado del servidor, siendo ideal para solicitudes de lectura de información

 Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

Significa que la solicitud ha sido exitosa. En términos simples, el código de respuesta 200 OK indica que la página web solicitada se ha cargado correctamente y que el servidor ha respondido de manera satisfactoria a la petición del cliente.

 ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

Sí, es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404. Un error 404 indica que la página solicitada no se encuentra en el servidor, lo que puede deberse a que la URL está mal escrita, la página ha sido eliminada o movida, entre otras razones.

 ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

Si, porque un desarrollador debe corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500. El error 500, también conocido como "Internal Server Error", indica un problema interno en el servidor que impide mostrar la página solicitada. Este error puede tener diversas causas, como errores en la configuración del servidor, problemas en el código del sitio web, sobrecarga del servidor, entre otros.

 ¿Qué componentes de estructura y estilo tiene una tabla?

Componentes de estructura

  1. table

    2. Define la estructura principal de la tabla.

  2. caption

    3. Proporciona una leyenda o título para la tabla.

  3. tr

    4. Representa una fila en la tabla.

  4. th

    5. Define una celda de cabecera en una fila.

  5. td

    6. Define una celda de datos en una fila.

Componentes de estilo

  1. Bordes

    2. Se establecen mediante propiedades de estilo para definir los bordes de la tabla y las celdas.

  2. Colores

    3. Se pueden aplicar estilos de color a la tabla y sus elementos para mejorar la apariencia visual.

  3. Alineación

    4. Permite alinear el contenido de las celdas horizontal y verticalmente

Sobre el ciclo de vida y desarrollo de los sistemas de información: ¿Cuál es el ciclo de vida de los sistemas de información? ¿Cuál es el ciclo de desarrollo de sistemas de información?

El ciclo de vida de los sistemas de información es un enfoque por fases del análisis y diseño que sostiene que los sistemas son desarrollados de mejor manera mediante el uso de un ciclo específico de actividades del analista y del usuario. Por otro lado, el ciclo de desarrollo de sistemas de información consta de varias etapas, que pueden variar ligeramente según el modelo utilizado, pero generalmente incluyen: planificación, análisis de requisitos, diseño y prototipo, desarrollo del software, pruebas, lanzamiento y mantenimiento.

Guerrero, A. C. (2015, July 20). Ciclo de vida de un sistema de información • gestiopolis. Gestiopolis. https://www.gestiopolis.com/ciclo-de-vida-de-un-sistema-de-informacion/
