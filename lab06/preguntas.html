Preguntas a responder

¿Por qué es una buena práctica usar JavaScript para checar que sean válidos los inputs de las formas antes de enviar los datos al servidor?

Respuesta: Porque mejora la experiencia del usuario con retroalimentación instantánea. Reduce la carga del servidor al filtrar datos incorrectos, optimiza el ancho de banda al evitar envíos innecesarios y mejora la seguridad al prevenir ataques simples. Proporciona control sobre mensajes de error y mejora la coherencia de la experiencia del usuario.

¿Cómo puedes saltarte la seguridad de validaciones hechas con JavaScript?

Respuesta: Deshabilitando o modificando el código JavaScript en el navegador utilizando herramientas de desarrollo como el Inspector de Chrome. Los usuarios pueden enviar directamente solicitudes al servidor sin pasar por el formulario en el navegador, lo que evade la validación del lado del cliente.

Si te puedes saltar la seguridad de las validaciones de JavaScript, entonces ¿por qué la primera pregunta dice que es una buena práctica?

Respuesta: Es una buena práctica utilizar JavaScript para validar los formularios porque proporciona una mejor experiencia al usuario al ofrecer retroalimentación instantánea y reducir la carga del servidor. Aunque se puede eludir la validación de JavaScript, esta sigue siendo una capa adicional de seguridad y comodidad para el usuario. Aunque no sea infalible, ayuda a prevenir errores comunes y mejora la eficiencia del sistema. Sin embargo, es crucial complementarla con validaciones en el servidor para una seguridad integral de los datos.
